子串

子串问题。

这类问题通常涉及对一个给定的字符串进行操作或分析，核心在于寻找满足特定条件的连续子串。

首先，明确一个关键概念：

  • 子串：必须是原字符串中连续的一段字符序列。
  • 子序列：则不要求连续，只需保持相对顺序即可。

例如，字符串 "abc"：

  • 子串有："a", "b", "c", "ab", "bc", "abc"。
  • 子序列除了包含所有子串，还有 "ac"。

核心思路与技巧

解决子串问题的核心方法论几乎都围绕一个关键技术：滑动窗口。

滑动窗口

1. 什么是滑动窗口？ 它使用两个指针（通常命名为 left 和 right）来维护一个窗口（一个连续的子串），通过动态地调整 left 和 right 指针，在字符串上滑动，从而高效地检查所有可能的候选子串。

2. 为什么有效？ 它避免了暴力解法（检查所有子串，O(n²) 时间复杂度）的重复计算，通常能将时间复杂度优化到 O(n)。

3. 两种主要类型：

  • 固定大小的窗口：窗口长度是固定的。先初始化一个窗口，然后每次同时将 left 和 right 向右移动一位。
    • 适用场景：求长度为 k 的子串的最大和、包含所有字符的最短子串等问题的变体。
  • 动态大小的窗口：这是更常见的形式。窗口大小不固定。
    • 扩张：right 指针向右移动，扩大窗口，直到窗口内的子串满足题目要求。
    • 收缩：一旦满足要求，left 指针向右移动，收缩窗口，试图找到更优的解或为下一次扩张做准备。
    • 适用场景：绝大多数求“最长”或“最短”满足条件的子串问题。

4. 辅助数据结构：哈希表 滑动窗口常常需要快速判断窗口内字符的频率或是否存在某个字符。这时通常会配合使用哈希表（在 Python 中是 dict 或 defaultdict，在 Java 中是 HashMap）来记录窗口内的字符及其出现次数。

常见子串题型及解题策略

以下是几类最经典的子串问题：

1. 最长无重复字符子串

  • 问题描述：给定一个字符串，请你找出其中不含有重复字符的 最长子串 的长度。
  • 示例："abcabcbb" 的答案是 3（"abc"）。
  • 解法：
    1. 使用滑动窗口 [left, right] 和哈希表 seen 记录字符最后一次出现的位置。
    2. right 向右移动，遍历每个字符。
    3. 如果当前字符 s[right] 已经在 seen 中且其位置 >= left（说明它在当前窗口内），则将 left 指针移动到 seen[s[right]] + 1，从而将重复字符排除出窗口。
    4. 更新 seen[s[right]] 的值为当前 right。
    5. 记录当前窗口长度 (right - left + 1) 并更新最大值。

2. 最小覆盖子串

  • 问题描述：给定一个字符串 S 和一个字符串 T，找出 S 中涵盖 T 所有字符的最短子串。
  • 示例：S = "ADOBECODEBANC", T = "ABC"，答案是 "BANC"。
  • 解法：
    1. 使用滑动窗口和两个哈希表：need 记录 T 中每个字符需要的次数，window 记录当前窗口内相关字符的数量。
    2. 用 valid 变量记录当前窗口中满足 need 要求的字符种类数。
    3. right 向右移动扩张窗口，将新字符加入 window。如果该字符在 need 中且数量达标，则 valid++。
    4. 当 valid == len(need)（即窗口已覆盖 T），收缩 left 指针。
    5. 在收缩过程中，记录最短子串的起始位置和长度。
    6. 将 left 指向的字符移出窗口，并更新 window 和 valid。

3. 找到字符串中所有字母异位词

  • 问题描述：给定一个字符串 s 和一个非空字符串 p，找到 s 中所有是 p 的字母异位词（排列） 的子串的起始索引。
  • 示例：s = "cbaebabacd", p = "abc"，答案是 [0, 6]（"cba" 和 "bac"）。
  • 解法：
    1. 这道题可以看作是固定大小窗口的问题，因为目标的异位词长度固定为 len(p)。
    2. 同样使用两个哈希表：need 记录 p 的字符频次，window 记录当前窗口的字符频次。
    3. 先初始化一个长度为 len(p) 的窗口。
    4. 比较 window 和 need 是否相等。如果相等，则记录起始索引 left。
    5. 然后窗口整体向右滑动一位：right++, left++，同时更新 window 哈希表，并再次比较。

4. 含有所有字符的最短/最长子串

  • 这是上述“最小覆盖子串”问题的泛化，模式完全一致。核心依然是扩张找可行解，收缩优化解。

5. 最大连续字符/数字子串

  • 这类问题有时可以不用哈希表，而是用滑动窗口模拟。
  • 示例：求最大连续 1 的个数（可能允许翻转 k 个 0）。
    • 窗口内允许最多有 k 个 0。当 0 的数量超过 k 时，收缩 left 指针，直到 0 的数量恢复 <= k。
class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        num = 0
        for left in range(len(nums)):
            summ = nums[left]
            if summ == k:
                num+=1
            for right in range(left+1,len(nums)):
                summ+=nums[right]
                if summ == k:
                    num+=1            
        return num

class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        dic = defaultdict(int)
        lis = []
        lis.append(0) #真乃点睛之笔，求和的列表加了一项0，则前零项和为0，这样可以解决下面right-left，left可以为0的情况
        num = 0
        summ = 0
        for i in nums:
            summ+=i
            lis.append(summ)

        for right in lis:
            num+=dic[right-k]
            dic[right]+=1
        return num
