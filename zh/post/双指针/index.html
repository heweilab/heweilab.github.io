双指针

双指针技巧的核心是使用两个指针（通常是索引或引用）以不同的策略遍历数据结构（主要是数组和链表），从而高效地解决问题。它能够将原本需要多重循环的暴力解法优化为线性或接近线性的时间复杂度（O(n) 或 O(n log n)），极大地提升效率。

双指针题型丰富，但主要可以分为以下几大类：

1. 左右指针（对撞指针）

这是最经典的双指针类型。两个指针分别位于序列的两端（一个在开头 left，一个在末尾 right），然后向中间移动，直到它们相遇或交错。

特点：

  • 通常用于有序数组（但并非绝对）。
  • 可以跳过许多不必要的枚举组合。

常见题型：

  1. 两数之和 II - 输入有序数组 (Two Sum II)
    • 问题： 在有序数组中找出两个数，使它们的和等于目标值。
    • 解法： left 指向最小数，right 指向最大数。
      • 如果 nums[left] + nums[right] < target，说明和太小，left 右移。
      • 如果 nums[left] + nums[right] > target，说明和太大，right 左移。
      • 相等时即为答案。
  2. 盛最多水的容器 (Container With Most Water)
    • 问题： 用数组表示一系列板子的高度，求哪两块板子能盛最多的水。
    • 解法： left 和 right 分别指向首尾。水量由 min(height[left], height[right]) * (right - left) 决定。
      • 每次移动高度较小的那个指针（因为移动高的指针只会让水量变得更小）。
  3. 三数之和 (3Sum) / 最接近的三数之和
    • 问题： 找出数组中所有不重复的三元组，使得和为 0。
    • 解法： 先排序。固定一个数 i，然后在其后面的区间使用左右指针 left 和 right 寻找 nums[left] + nums[right] == -nums[i]。通过移动指针来跳过重复解。
  4. 验证回文串 (Valid Palindrome)
    • 问题： 判断一个字符串是否是回文串（忽略大小写和非字母数字字符）。
    • 解法： left 从头开始，right 从尾开始，向中间移动。跳过非字母数字字符，判断 s[left] 和 s[right] 是否相等（忽略大小写）。

2. 快慢指针

两个指针从同一侧开始，以不同的速度前进。快指针fast的移动速度是慢指针slow的两倍或更快。

特点：

  • 主要用于链表相关的问题。
  • 常用于检测循环或寻找特定节点。

常见题型：

  1. 判断链表是否有环 (Linked List Cycle)
    • 问题： 判断一个链表中是否存在环。
    • 解法： slow 每次走一步，fast 每次走两步。如果链表中存在环，fast 最终会追上 slow（相遇）。如果 fast 走到尽头（null），则无环。
  2. 寻找链表的中间结点 (Middle of the Linked List)
    • 问题： 找到链表的中间节点。
    • 解法： slow 每次走一步，fast 每次走两步。当 fast 走到链表末尾时，slow 正好在中间。
  3. 寻找链表的倒数第 k 个结点
    • 问题： 找到链表中倒数第 k 个节点。
    • 解法： 让 fast 指针先走 k 步，然后 slow 和 fast 以相同速度前进。当 fast 走到末尾时，slow 就在倒数第 k 个位置。
  4. 寻找环的入口节点
    • 问题： 如果链表有环，找到环开始的位置。
    • 解法： 先使用快慢指针找到相遇点。然后将一个指针重置到链表头，两个指针以相同速度前进，再次相遇的点即为环的入口。

3. 滑动窗口

这是双指针的一种高级形式，两个指针 left 和 right 形成一个窗口 [left, right]，通过移动 right 来扩大窗口，移动 left 来缩小窗口，从而在窗口内寻找满足条件的区间。

特点：

  • 主要用于解决连续子数组/子串的相关问题。
  • 目标是找到满足某些条件的最短/最长/最值的子区间。

常见题型：

  1. 无重复字符的最长子串 (Longest Substring Without Repeating Characters)
    • 问题： 给定一个字符串，请你找出其中不含有重复字符的 最长子串 的长度。
    • 解法： 用 left 和 right 维护一个窗口。用 Set 或 Map 记录窗口内的字符。right 右移扩大窗口，如果新字符已存在，则 left 右移缩小窗口，直到窗口中无重复字符。
  2. 最小覆盖子串 (Minimum Window Substring)
    • 问题： 在字符串 S 中找出包含字符串 T 所有字符的最小子串。
    • 解法： right 右移直到窗口包含 T 的所有字符，然后 left 右移尝试收缩窗口，找到能满足条件的最小窗口。记录最小值，然后继续移动 right 寻找下一个可行解。
  3. 长度最小的子数组 (Minimum Size Subarray Sum)
    • 问题： 给定一个含有 n 个正整数的数组和一个正整数 target，找出该数组中满足其和 ≥ target 的长度最小的 连续子数组。
    • 解法： right 右移累加和，当和 >= target 时，记录长度并尝试 left 右移，看是否能缩小窗口但仍满足条件。

4. 分离指针（用于多个数据结构）

两个指针分别遍历两个不同的数组或链表。

特点：

  • 用于处理多个序列的问题。

常见题型：

  1. 合并两个有序数组 (Merge Sorted Array)
    • 问题： 将两个有序数组合并到第一个数组中。
    • 解法： 从两个数组的末尾开始（p1 在 nums1 的有效元素尾，p2 在 nums2 的尾），比较大小，将较大的元素放到 nums1 的最终末尾（从后往前填充），避免覆盖。
  2. 两个数组的交集 (Intersection of Two Arrays)
    • 问题： 给定两个数组，编写一个函数来计算它们的交集。
    • 解法（已排序数组）： 先对两个数组排序。然后用指针 i 和 j 分别遍历两个数组。根据 nums1[i] 和 nums2[j] 的大小关系移动相应的指针，相等时则记录结果。

总结与解题技巧

类型 指针初始位置 移动方向 典型应用 关键点
左右指针 首尾 向中间夹逼 有序数组的和、容器盛水 利用有序性排除无效解
快慢指针 同一起点 一快一慢 链表环、中点、倒数第k个 速度差产生奇妙关系
滑动窗口 同一起点 一前一后 连续子串/子数组问题 维护窗口的合法性
分离指针 不同结构起点 各自独立移动 合并有序数组、求交集 同步遍历多个结构

如何识别双指针问题？

  • 问题涉及数组或链表。
  • 要求寻找连续子区间、配对、顺序关系。
  • 暴力解法是 O(n²) 或 O(n³) 的，可能可以通过双指针优化到 O(n) 或 O(n log n)。

283. 移动零

已解答 简单

提示

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

请注意 ，必须在不复制数组的情况下原地对数组进行操作。

示例 1:

输入: nums = [0,1,0,3,12]
输出: [1,3,12,0,0]

示例 2:

输入: nums = [0]
输出: [0]

提示:

  • 1 <= nums.length <= 104
  • -231 <= nums[i] <= 231 - 1

进阶：你能尽量减少完成的操作次数吗？

解题代码：

一开始的做题思路其实很简单，就是利用 Python List 的 pop 和 append 函数，遍历、删除、添加；思路很简单，也有点取巧了。因为过程当中要考虑 nums 的改写，所以过程也遇到了问题，比如：如果直接在遍历的时候删除，会导致 i 的索引错位的问题，这个时候删除一个添加一个，很难找到错位的规律，那就只在遍历的时候考虑末尾增加，用空 List 记录删除位点，重新进行一次删除，同样在重新删除的时候要把错位考虑进去，这个时候每次删除，错位-1即可。

这个时候其实是没有考虑到双指针的知识点的，但是后来回顾发现，用空列表保存0位点，好像还和双指针沾点边。

class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        N = []
        n = 0
        for i in range(len(nums)-1):
            if nums[i] == 0:
                N.append(i)
                nums.append(0)
        for j in N:
            nums.pop(j-n)
            n+=1

看了网上的解析，很多用双指针来求解，解法很多，我看懂了一个比较好理解的，代码如下：

class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        left = 0
        right = 0
        for i in range(len(nums)):
            if nums[right] != 0:
                nums[right],nums[left] = nums[left],nums[right]
                left+=1
            right+=1

解析：从左往右依次遍历，把所有非0的数据依次提到最前，即可求解。一个指针用于遍历，一个指针用于记录下一个需要替换的值的位置。

11. 盛最多水的容器

已解答 中等

给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

**说明：**你不能倾斜容器。

示例 1：

输入：[1,8,6,2,5,4,8,3,7]
输出：49 
解释：图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。在此情况下，容器能够容纳水（表示为蓝色部分）的最大值为 49。

示例 2：

输入：height = [1,1]
输出：1

提示：

  • n == height.length
  • 2 <= n <= 105
  • 0 <= height[i] <= 104

解题代码：

滑动窗口的问题，左右指针往中间靠拢，找到比之前更大的面积时进行替换。这里有一个如何滑动的选择：因为求面积，只需要比较左右边界的高度，滑动矮的那个即可，另外我这里加了一个判断，判断滑动后的高度只有大于原高度时，才进行计算判断。

class Solution:
    def maxArea(self, height: List[int]) -> int:
        left = 0
        right = len(height)-1
        L = min(height[left],height[right])*(right-left)
        while left<right:
            if height[left]>height[right]:
                right-=1
                if height[right]>height[right+1]:
                    L = max(min(height[left],height[right])*(right-left),L)
            else:
                left+=1
                if height[left]>height[left-1]:
                    L = max(min(height[left],height[right])*(right-left),L)
        return L

如果滑动一次就计算一次的话，代码比较简单，但是耗时稍微长一些：

class Solution:
    def maxArea(self, height: List[int]) -> int:
        left = 0
        right = len(height)-1
        L = 0
        while left<right:
            result = min(height[left],height[right])*(right-left)
            L = max(result,L)
            if height[left]>height[right]:
                right-=1            
            else:
                left+=1
        return L

15. 三数之和

已解答 中等

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

**注意：**答案中不可以包含重复的三元组。

示例 1：

输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。

示例 2：

输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0 。

示例 3：

输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0 。

提示：

  • 3 <= nums.length <= 3000
  • -105 <= nums[i] <= 105

解题代码：

思路：借鉴了LeetCode官方的解析，通过排序+相邻两次枚举的元素不能相同的方法满足“不重复”的要求。直观的解法是固定前两个数，直接搜索第三个数然后赋值输出，不出意外，超时了。

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums = sorted(nums)
        result = []
        for i in range(len(nums)-2):
            if nums[i]!=nums[i-1] or i==0:
                for j in range(i+1,len(nums)-1):
                    if nums[j]!=nums[j-1] or j-1==i:
                        three = 0-(nums[i]+nums[j])
                        if three in nums[j+1:]:
                            result.append([nums[i],nums[j],three])
        return result

这里我的解决办法是限制第三个数寻找的下界限，因为排序的原因，每次循环第二个数，总会变大， 如果存在第三个数的解的话，一定是在上一个解的左边出现，因此我这里是想保留上一次出现解的第三个数的位置，来当做下次寻找的下界限。（解题思路其实类似哈希里的两数之和，但不同的是那里一定有且仅有一个解，这里不太一样）

但是这样做仍然有问题：①存在无解的情况，如果不存在第三个数的解，那么还是每次都得全部遍历一遍去找第三个数（而且找不到）。②这种寻找方式还是从第二个数的下一个为起始点往后寻找，这样仍然在每次重复寻找步骤，增加时间复杂度。

所以过不了特殊的测试用例，超时。

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums = sorted(nums)
        index_map = {}
        for idx, value in enumerate(nums):
            index_map[value] = idx
        result = []
        l = len(nums)
        for i in range(l-2):
            if nums[i]!=nums[i-1] or i==0:
                for j in range(i+1,l-1):
                    if nums[j]!=nums[j-1] or j-1==i:
                        if l>j+1:
                            three = 0-(nums[i]+nums[j])
                            if three in nums[j+1:l]:
                                l = index_map[three]
                                result.append([nums[i],nums[j],three])
            l = len(nums)
        return result

这是参考官方解题思路修改来的，这个思路很巧妙的是：确定第一个数，用双指针从前往后、从后往前寻找目标组合，而且它的判断不是定向寻找，而且界限划定，什么意思呢？就是说如果不存在解的集合，也能够对第三个数的下界限进行一个更新（更新到最后一个大于0的位置，下次从这个位置往左寻找，原理和上面相同，但是判定条件不再是==而是>）。增加的 if j == third: break 甚至能确定后续第二个数的无解，进一步提高了处理效率。

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums = sorted(nums)
        result = []
        l = len(nums)
        for i in range(l-2):
            third = l-1
            if nums[i]!=nums[i-1] or i==0:
                for j in range(i+1,l-1):
                    if nums[j]!=nums[j-1] or j-1==i:
                        while j<third and nums[j]+nums[third]+nums[i]>0:
                            third-=1
                        if j == third:
                            break
                        if nums[j]+nums[third]+nums[i] == 0:
                            result.append([nums[i],nums[j],nums[third]])
        return result
